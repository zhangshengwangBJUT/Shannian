# 闪念 (Shannian)

> 用自然语言代替繁琐的拖拽搭积木，让 AI 帮你设计效率动作，一键自动化执行。

## 这是什么？

闪念是一个 Windows 桌面效率工具，核心思路是 **AI 做设计师，传统引擎做执行者**。

你用自然语言描述需求，AI 理解意图并生成动作配置，之后这个动作由本地引擎零延迟执行。AI 只参与"创建动作"这一次，不参与每次执行。

## 和现有工具有什么不同？

| 维度 | 通用 AI Agent | 传统效率工具 (Quicker/AHK) | 闪念 |
|------|--------------|--------------------------|------|
| 使用门槛 | 低，自然语言交互 | 高，需学脚本/搭积木 | 低，自然语言交互 |
| 执行速度 | 2-5 秒（每次推理） | 毫秒级 | 毫秒级 |
| 结果确定性 | 不确定，同一指令行为可能不同 | 确定 | 确定 |
| AI 依赖 | 每次执行都依赖 | 无 | 仅创建时依赖 |

一句话总结：**闪念用 AI 解决"搭动作难"的问题，用传统引擎解决"执行慢且不确定"的问题。**

## 核心功能

- **自然语言输入** — 全局热键 (Ctrl+Space) 呼出输入框，说话代替拖拽
- **AI 意图识别** — 理解需求，自动拆解成可执行步骤
- **动态追问** — 信息不足时主动提问澄清
- **执行计划确认** — 展示步骤列表，用户确认或修改后执行
- **动作固化** — 常用动作保存为模板，下次一键触发，零延迟

## 安装

1. 前往 [Releases](../../releases) 下载 `Shannian-Windows-Portable-0.1.1.zip`
2. 解压到任意目录
3. 运行 `闪念.exe`，免安装

**系统要求**：Windows 10/11，需联网（AI 意图识别）

## 使用流程

```
按下 Ctrl+Space 呼出输入框
        ↓
用自然语言描述需求（如"翻译选中文本并复制"）
        ↓
AI 理解意图，必要时追问澄清
        ↓
AI 展示执行步骤，用户确认
        ↓
引擎自动执行，返回结果
        ↓
可保存为常用动作，下次一键触发
```

## 技术栈

- **前端**：Electron + React
- **后端**：Python FastAPI + LLM Agent
- **模型**：Claude Sonnet（意图识别 + 工具调用）
- **打包**：electron-builder
